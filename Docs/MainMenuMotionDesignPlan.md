# 《负一和一》主界面动态化设计方案

更新时间：2026-06-11  
目标：将当前静态主界面升级为动态标题页，并为卡牌、棋子、筹码设计专属动效。  
结论：**主要可以在 Unity 中完成；外部软件只在需要“重绘、分层、逐帧、骨骼动画、复杂特效贴图”时辅助。**

## 1. 总体判断

当前主界面已经有高质量静态背景。下一阶段不建议删除卡牌、棋子、筹码，而是把它们从“静态贴片”升级成“动态桌游符号”。

推荐方向：

> 背景轻微呼吸 + 云雾漂移 + 卡牌悬浮翻动 + 棋子轻微对峙 + 筹码微光旋转

重点是“微动态”，不是夸张动画。主菜单应该像一张活着的标题插画，而不是战斗场景。

## 2. Unity 能完成什么

| 效果 | Unity 可完成度 | 推荐方式 |
| --- | --- | --- |
| 背景慢速平移/视差 | 高 | 多层 Image / RawImage UV 滚动 / RectTransform 动画 |
| 云雾漂移 | 高 | 半透明云雾贴图 + 粒子系统 / Shader UV Offset |
| 日光呼吸 | 高 | UI Image alpha 动画 / Light 2D / Shader |
| 卡牌漂浮 | 高 | RectTransform 位移、旋转、缩放循环 |
| 卡牌翻动 | 中高 | 2D 缩放 X 轴模拟 / Shader / Sprite Swap |
| 棋子轻微摇摆 | 高 | 旋转、缩放、位移循环 |
| 筹码旋转闪光 | 高 | Z 轴旋转 / 高光扫过遮罩 |
| 鼠标悬停菜单反馈 | 高 | Button transition / Animator / DOTween |
| 粒子光点 | 高 | Unity Particle System |
| 复杂角色立绘动效 | 中 | 需要 Spine/Live2D/序列帧辅助 |
| 背景画面分层重绘 | 低到中 | 需要 Photoshop/Krita 等外部软件 |

## 3. 什么时候需要其他软件

Unity 可以做大部分“运动”，但做不了所有“素材生产”。是否需要外部软件取决于你想要的完成度。

### 3.1 不需要外部软件的情况

如果目标是 Demo 级动态标题页：

- 背景整体微动
- 云雾轻微漂移
- 卡牌上下浮动
- 棋子轻微晃动
- 筹码小幅旋转
- 菜单 hover 高亮
- 光点粒子

这些都可以直接在 Unity 中完成。

### 3.2 建议外部软件辅助的情况

如果目标是更高级的动态封面：

| 需求 | 推荐软件 | 原因 |
| --- | --- | --- |
| 把背景拆成前景/中景/远景多层 | Photoshop / Krita / Photopea | 需要修补被分离后露出的背景。 |
| 画新的专属卡牌牌面 | Photoshop / Krita / Clip Studio Paint | Kenney 资源风格偏扁平，专属牌面会更统一。 |
| 做逐帧卡牌翻转 | Aseprite / Photoshop / After Effects | 可输出序列帧给 Unity。 |
| 做复杂 2D 骨骼角色 | Spine / Live2D / DragonBones | Unity 原生不适合高质量角色骨骼动画生产。 |
| 做复杂粒子贴图 | Photoshop / Krita | 粒子本体可在 Unity 做，但贴图需要外部绘制。 |

## 4. 推荐技术路线

### 路线 A：Unity 内完成，推荐当前 Demo 使用

目标：成本低、风险低、能快速出效果。

实现内容：

1. 背景作为底层 Image，做极慢缩放或位移。
2. 增加 2-3 层半透明云雾 Image，缓慢横向漂移。
3. 右侧卡牌使用 RectTransform 动画：上下浮动、轻微旋转。
4. 棋子使用小幅摇摆动画。
5. 筹码使用慢速旋转和 alpha 呼吸。
6. 菜单文字 hover 时轻微右移、变亮。
7. 加少量金色/青色粒子光点。

优点：

- 不依赖新工具。
- 修改快。
- 易在 Unity 中验证。
- 不影响发布包体太多。

缺点：

- 背景仍然是单图，视差有限。
- 卡牌/棋子如果素材本身不够高级，只靠动效无法完全解决风格差异。

### 路线 B：外部拆图 + Unity 动态，推荐后续优化

目标：让静态背景真正有纵深。

实现内容：

1. 外部软件把主界面背景拆成：
   - 远景天空/日光
   - 中景宫殿/山峰
   - 前景竹叶/左下棋盘/右下塔阁
   - 云雾层
2. Unity 中用不同层级做慢速视差。
3. 前景竹叶轻微摆动。
4. 云雾层循环漂移。
5. 宫殿区域加淡光呼吸。

优点：

- 画面高级感明显提升。
- 更接近商业游戏动态标题页。

缺点：

- 需要拆图和补图。
- 工时更高。
- 如果拆图粗糙，会出现边缘穿帮。

### 路线 C：重做专属主视觉，暂不建议

目标：制作完全定制的角色/卡牌/棋局动态封面。

不建议当前做，原因：

- 成本高。
- 会偏离当前 Demo 收尾节奏。
- 需要新的美术产出标准。

## 5. 卡牌、棋子、筹码动效设计

### 5.1 卡牌动效

定位：身份与谎言。

建议效果：

| 动作 | 参数 |
| --- | --- |
| 上下浮动 | 8-16 px，周期 3-5 秒 |
| 轻微旋转 | -3° 到 +3°，周期 4-6 秒 |
| 前后错位 | 两张牌不同步运动 |
| 边缘微光 | 红/绿低透明光晕，alpha 0.15-0.35 |
| 偶发翻面 | 每 8-12 秒一次，或只在进入主菜单时播放一次 |

Unity 实现：

- Animator
- Coroutine + Mathf.Sin
- DOTween（如果项目愿意引入）
- 自写 `MenuFloatMotion` 脚本

注意：

- 翻牌不要太频繁，否则会抢菜单注意力。
- 卡牌不要使用过强实体边框，建议做半透明牌影或专属牌面。

### 5.2 棋子动效

定位：阵营对峙。

建议效果：

| 动作 | 参数 |
| --- | --- |
| 轻微摇摆 | -2° 到 +2° |
| 呼吸缩放 | 0.98-1.03 |
| 入场落子 | 主菜单打开时从上方落下 20-40 px，回弹一次 |
| 对峙光点 | 绿色/红色小光点在棋子旁缓慢闪烁 |

注意：

- 棋子可以小，但要有“落在棋局上”的感觉。
- 不建议让棋子一直大幅移动。

### 5.3 筹码动效

定位：分数与猜分。

建议效果：

| 动作 | 参数 |
| --- | --- |
| 慢速旋转 | 5-15° 循环，不要完整高速转圈 |
| 高光扫过 | 低透明金色条从左到右扫过 |
| 堆叠轻震 | 进入界面时轻微抖一下 |

注意：

- 筹码不再作为右侧主视觉，可放在左下棋盘或菜单附近作为点缀。
- 数量少，避免像休闲赌博游戏。

## 6. 主界面背景动效设计

### 6.1 单图动态，当前推荐

可以在 Unity 中直接做：

| 动效 | 做法 |
| --- | --- |
| 背景慢缩放 | 背景 Image 从 1.00 到 1.025 缓慢呼吸 |
| 云雾漂移 | 叠加 2 层半透明云雾图，循环横移 |
| 日光呼吸 | 右上光晕 Image alpha 在 0.12-0.22 间变化 |
| 竹叶前景 | 如果可单独切出，做轻微旋转摆动 |
| 粒子光点 | Particle System，少量、慢速、低透明 |

### 6.2 分层动态，后续推荐

如果愿意外部拆图：

| 层 | 动效 |
| --- | --- |
| 天空远景 | 极慢横移 |
| 远山/宫殿 | 几乎不动，仅光照呼吸 |
| 云雾 | 横向漂移 + alpha 变化 |
| 左上竹叶 | 轻微摆动 |
| 左下棋盘 | 不动，作为视觉锚点 |
| 右下塔阁 | 不动或轻微前景视差 |

## 7. 推荐落地顺序

### Phase 1：Unity 微动态

1. 菜单文字 hover/选中动效。
2. 背景轻微呼吸。
3. 云雾漂移。
4. 卡牌浮动。
5. 棋子落子/摇摆。
6. 筹码微光。

验收：

- 主界面看起来“活着”。
- 不影响按钮点击。
- 动效不让玩家分心。

### Phase 2：素材质感修正

1. 自制更统一的卡牌轮廓或牌面。
2. 弱化 Kenney 素材贴片感。
3. 给卡牌、棋子、筹码加统一阴影和色调。

验收：

- 卡牌/棋子/筹码不再像临时 UI 素材。
- 与背景色调统一。

### Phase 3：外部拆图视差

1. 拆分背景层。
2. 修补背景空洞。
3. Unity 中做视差和云雾。

验收：

- 主界面有纵深。
- 没有明显切图边缘。

## 8. 技术建议

### 8.1 不引入第三方库的实现

可写一个通用脚本：

```text
MenuLoopMotion
- positionAmplitude
- rotationAmplitude
- scaleAmplitude
- duration
- phaseOffset
- useUnscaledTime
```

用于卡牌、棋子、筹码、菜单装饰。

优点：

- 简单。
- 无外部依赖。
- 适合当前项目。

### 8.2 可以考虑 DOTween，但不是必须

DOTween 能提高 UI 动画开发效率，但当前项目没有必要为了主菜单强行引入依赖。若已有团队熟悉 DOTween，可以使用；否则原生 Animator/Coroutine 足够。

### 8.3 粒子建议

使用 Unity Particle System：

- 少量粒子。
- 低速度。
- 低透明度。
- 青色/金色为主。
- 不要覆盖按钮区域。

## 9. 是否需要其他软件

明确回答：

| 目标 | 是否需要其他软件 |
| --- | --- |
| Demo 级动态主菜单 | 不需要，Unity 可以完成。 |
| 卡牌/棋子/筹码简单动态图 | 不需要，Unity 可以完成。 |
| 更统一的专属卡牌牌面 | 建议使用 Photoshop/Krita/Photopea。 |
| 背景分层视差 | 建议使用 Photoshop/Krita/Photopea 拆图补图。 |
| 高质量角色动态立绘 | 需要 Spine/Live2D/After Effects 等。 |

推荐当前选择：

> 先用 Unity 做微动态，再判断是否值得外部拆图和重绘卡牌。

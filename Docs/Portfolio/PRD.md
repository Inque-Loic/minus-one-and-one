# 《负一和一》产品需求文档（PRD）

更新日期：2026-06-10

发布状态：Itch 已上传，GitHub 仓库与 Release 已公开

## 1. 项目定位

《负一和一》是一款面向线下桌游/社交推理场景的轻量电子游戏原型。玩家被分为“一阵营”和“负一”，通过接触、拒绝、公开分数变化、指控与信任来影响分数，并在有限回合内判断谁更可能属于对立阵营。

当前 Demo 的目标不是做复杂包装，而是验证核心机制是否成立：

- 玩家是否能理解身份、目标和行动。
- AI 行为是否能稳定跑完整局。
- 分数、阵营胜负和猜分摘要是否能让结算结果可理解。
- Windows 包是否能被第三方直接下载运行。

## 3. 明确不做的内容

为保持 Demo 简洁，当前版本不做完整“终局复盘”系统，也不做复杂时间线回放、接触链图谱或逐回合推理解释。已有文档中与“终局复盘”相关的后续目标应视为已取消。

当前保留的结算信息只包括：

- 阵营胜负。
- 玩家身份、分数和排名。
- 猜分结果摘要。

## 5. 平衡测试基线

测试来源：Unity 测试模式，批量 1000 局。

| 指标 | 数值 |
| --- | --- |
| 参数版本 | Docs/Portfolio CSV |
| 人数 | 8 |
| 负一人数 | 2 |
| 回合数 | 7 |
| AI 延迟 | 0.01s |
| 样本 | 1000 局 |
| 一阵营胜 | 580（58.0%） |
| 负一胜 | 355（35.5%） |
| 平局 | 65（6.5%） |
| 平均分 | 一阵营 3.20 / 负一 2.73 |
| 接触 | 25651 次 |
| 接受率 | 72.1% |
| 拒绝 | 7149 次 |
| 放弃 | 6489 次 |
| 猜分参与率 | 80.3% |
| 平均猜对 | 1.51 人 |

当前结论：胜率偏向一阵营，但没有阻塞发布。该数据适合作为 v0.1.1 Demo 的首个公开平衡基线。

## 7. 成功标准

v0.1.1 Demo 已达到以下标准：

- 第三方可以下载并运行 Windows 版本。
- 手动完整局可玩通。
- 1000 局自动测试不卡死。
- 中文显示不再依赖系统商业字体。
- 规则和测试数据可通过 Portfolio 文档解释。

下一阶段的成功标准是让第一次打开游戏的玩家在不读外部文档的情况下，也能知道自己要做什么。

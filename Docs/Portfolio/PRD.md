# 《负一和一》PRD

## 1. 产品概述

《负一和一》是一款类狼人杀的轻量社交推理游戏。玩家被分为“一”和“负一”两个阵营，在有限回合内通过两两接触改变分数，并通过讨论、声明和终局猜分判断谁在说谎。Demo 版本以单机人机局为目标，验证核心规则、AI 行为和可读性，再决定是否进入多人联网版本。

一句话定位：用分数变化替代死亡淘汰，让每一次接触都成为一次可被推理、可被伪装、可被追责的风险交易。

## 2. 设计目标

| 目标 | 说明 | Demo 验收方式 |
| --- | --- | --- |
| 低门槛理解 | 玩家知道自己身份后，能在 1 分钟内理解“接触会改变分数，坏人会污染分数”。 | 首局无需阅读长教程即可完成 3 回合。 |
| 高信息密度 | 每回合至少产生一次可讨论的信息：谁接触谁、是否拒绝、本回合是否有坏人行动、分数声明。 | 回合日志和讨论消息能支持复盘。 |
| 可调平衡 | 阵营人数、回合数、初始分、AI 决策和怀疑度都可配置。 | 配置表驱动或集中参数面板可调整。 |
| 单机可演示 | 没有真人联机时，也能通过 AI 展示完整规则闭环。 | AI 可自动跑完 100 局并输出胜率。 |
| 作品集可解释 | 文档能说明规则推导、竞品参照、测试方法和迭代计划。 | PRD、配置表、竞品分析、开发计划齐全。 |

## 3. 目标用户

| 用户 | 需求 | 本项目对应点 |
| --- | --- | --- |
| 社交推理玩家 | 想要推理、欺骗、观察行为，但不一定想被早早淘汰。 | 分数制替代死亡淘汰，所有玩家持续参与。 |
| 桌游/派对游戏爱好者 | 喜欢短局、规则明确、容易复盘的局内故事。 | 7 回合结构、回合接触、讨论发言、终局猜分。 |
| 游戏策划评审/招聘方 | 想看到设计从规则到表格、测试、落地的完整链路。 | 配置表、AI 模拟、里程碑、竞品分析。 |

## 4. 核心玩法循环

1. 开局随机身份：多数玩家为“一”，少数玩家为“负一”。
2. 每回合按顺序行动：当前玩家选择一名可接触对象，或在规则允许时放弃。
3. 目标响应：目标可以接受或拒绝，连续不行动会被限制。
4. 接触结算：
   - 一与一接触：双方各 +1。
   - 一与负一接触：一方 -1，负一方 +1。
   - 负一与负一接触：不改变分数，但会触发“坏人行动”信息。
5. 讨论阶段：每名玩家发布一次声明、指控或辩护。
6. 终局猜分：玩家可猜测所有人的最终分数，猜得最好者加分，猜得最差者扣分。
7. 最终结算：展示身份、分数和排名，用于复盘。

## 6. 规则细节

### 6.1 玩家参数

| 参数 | 当前值 | 设计意图 |
| --- | --- | --- |
| 总人数 | 8 | 兼顾信息量与 UI 可读性。 |
| 负一人数 | 2 | 形成少数阵营协作与伪装压力。 |
| 最大回合 | 7 | 保证每人有多次行动机会，同时控制单局时长。 |
| 一初始分 | 1 | 好人初始正收益，鼓励接触。 |
| 负一初始分 | -1 | 负一开局低分，需要通过伪装接触追分。 |

### 6.2 接触限制

| 限制 | 当前规则 | 目的 |
| --- | --- | --- |
| 不能接触自己 | 是 | 避免无意义行为。 |
| 同一局不能重复接触同一对象 | 是 | 扩大信息网络，减少固定搭档刷分。 |
| 每回合每人最多参与一次接触 | 是 | 控制信息量和行动公平性。 |
| 放弃/拒绝计入不行动 | 是 | 避免玩家连续规避风险。 |
| 连续不行动限制 | 已有 `noContactLastRound` | 强制玩家进入风险选择。 |

### 6.3 讨论消息

| 类型 | 信息内容 | 系统影响 |
| --- | --- | --- |
| 声明分数 | 声称本回合分数变化 | 影响其他玩家对发言人的怀疑度。 |
| 指控 | 指定目标为坏人 | 提升目标怀疑度。 |
| 辩护 | 指定目标为好人 | 降低目标怀疑度。 |

### 6.4 胜负/表现判定

Demo 当前以最终分数排名展示个人表现；测试模式中用好人平均分与负一平均分比较阵营胜负。后续建议明确两层目标：

| 层级 | 判定 | 原因 |
| --- | --- | --- |
| 阵营胜负 | 好人平均分 > 负一平均分，则一阵营胜；反之负一胜。 | 用于平衡测试和版本调参。 |
| 个人表现 | 最终分数排名 + 猜分奖励 | 保留个体目标，提高每局参与动机。 |

## 8. UX 要求

| 场景 | 要求 |
| --- | --- |
| 身份页 | 第一眼明确“你是谁、你的目标、你该做什么”。 |
| 行动页 | 当前回合、当前行动者、可选目标、不可选原因必须清晰。 |
| 响应页 | 玩家知道对方是谁、接受/拒绝的后果、是否因连续不行动被限制。 |
| 讨论页 | 发言顺序、发言内容、目标对象和已发言记录可读。 |
| 猜分页 | 输入成本低，默认值、跳过、提交反馈明确。 |
| 结算页 | 不只给排名，还能解释“为什么输赢”。 |

## 9. 数据与测试指标

| 指标 | 目标区间 | 用途 |
| --- | --- | --- |
| 单局时长 | 8-15 分钟 | 控制 Demo 节奏。 |
| 好人胜率 | 45%-55% | 判断身份比例和分数规则是否基础平衡。 |
| 负一平均接触成功次数 | 每局 2-5 次 | 判断伪装空间。 |
| 玩家被迫行动次数 | 每局 1-3 次 | 判断连续不行动限制是否有效。 |
| 猜分参与率 | AI 30%-70%，人类不强制 | 判断终局机制是否过重。 |

## 10. 主要风险

| 风险 | 表现 | 应对 |
| --- | --- | --- |
| 信息过少 | 玩家只是在点按钮，不知道如何推理。 | 增加回合日志、分数变化提示、讨论可追踪线索。 |
| 信息过多 | 新手看不懂状态和结算。 | 分层展示：行动页只给必要信息，复盘页再展开。 |
| AI 行为太随机 | 批量测试胜率不可解释。 | 将 AI 决策拆成表格参数，并记录每次决策原因。 |
| 坏人收益过高 | 负一接触好人即可稳定滚分。 | 调整初始分、回合数、拒绝概率、终局猜分权重。 |
| UI 乱码/遮挡 | Demo 观感受损。 | 统一 UTF-8 编码，进行双分辨率走查。 |

# 类狼人杀竞品分析

调研时间：2026-06-08  
调研目的：提炼社交推理游戏中“信息产生、欺骗表达、局后复盘、可扩展配置”的成熟做法，并反推《负一和一》的 Demo 差异化。

## 1. 竞品选择

| 产品 | 类型 | 选择原因 |
| --- | --- | --- |
| Among Us | 行动 + 会议投票型社交推理 | 大众化入口清晰，任务/击杀/会议构成强节奏闭环。 |
| Goose Goose Duck | 行动 + 语音 + 多阵营角色 | 强调近距离语音、地图、角色、模式的内容扩展。 |
| Town of Salem 2 | 文本讨论 + 多角色策略 | 角色数量大，日夜循环和聊天推理承载深度。 |
| Blood on the Clocktower | 桌游主持人 + 角色脚本 | 信息设计极强，死亡后仍参与，局后故事性强。 |

## 2. 横向对比

| 维度 | Among Us | Goose Goose Duck | Town of Salem 2 | Blood on the Clocktower | 《负一和一》机会 |

| 核心信息源 | 路径、尸体、任务、会议发言 | 路径、语音、尸体、角色行为 | 夜间行动、聊天、遗言、投票 | 角色能力、公开处决、主持人反馈 | 接触对象、分数变化、声明真伪、猜分结果 |

## 3. 关键启发

### 3.1 信息必须可被追踪

社交推理不是“玩家自己想象出信息”，而是系统每回合稳定产出可讨论线索。竞品常见的信息载体是尸体、任务、夜间行动、角色能力。《负一和一》的信息载体应明确为：

1. 谁主动接触了谁。
2. 对方是否接受。
3. 本回合是否出现负一行动。
4. 每名玩家声称自己的分数变化。
5. 终局猜分和真实分数的差距。

因此 Demo 必须有回合日志和终局复盘，否则系统亮点会被 UI 流程吞掉。

### 3.2 分数制是差异化，不应只做成排名

传统狼人杀的紧张感来自死亡和投票；本项目的紧张感来自“接触后收益是否可信”。分数不能只是结算数字，而应该成为玩家推理的主材料：

| 设计点 | 建议 |
| --- | --- |
| 分数变化 | 每回合只展示玩家自己的真实变化，公共信息只展示“是否有负一行动”。 |
| 谎报空间 | 讨论阶段允许玩家声明任意 -2 到 +2，而非只能报真实值。 |
| 记忆负担 | 终局复盘展示每回合声明，降低玩家学习成本。 |
| 奖惩 | 猜分奖励让认真记录和识别谎言的玩家获得收益。 |

### 3.3 Demo 不应急着堆角色

Goose Goose Duck 和 Town of Salem 2 的内容量很大，但它们已有成熟社区和在线对局环境。当前项目在 Demo 阶段更适合证明基础规则的可玩性：

1. 先稳定“一”和“负一”的基础博弈。
2. 再加 1 个信息型角色或 1 个干扰型角色做扩展展示。
3. 每新增一个角色，都必须进入配置表和批量测试。

### 3.4 死亡后仍参与的思路值得吸收

Blood on the Clocktower 的重要启发是：社交推理游戏不必让失败者沉默。《负一和一》天然没有死亡淘汰，应把这点包装成核心卖点：

| 传统痛点 | 本项目表达 |
| --- | --- |
| 早死玩家无事可做 | 分数制全员参与到终局。 |
| 新手不敢发言 | 声明分数/指控/辩护提供结构化发言按钮。 |
| 局后只记得谁赢 | 复盘展示接触链和谎言链。 |

## 4. 本项目差异化定位

| 方向 | 表述 |
| --- | --- |
| 机制差异 | 用“两两接触 + 分数污染”替代“夜晚击杀 + 白天投票”。 |
| 体验差异 | 玩家不出局，所有人都持续参与收益、风险和推理。 |
| 策划差异 | 通过 AI 批量模拟调参，展示系统策划的数据闭环。 |
| Demo 差异 | 单机也能完整演示社交推理规则，而不依赖多人在线环境。 |

## 5. 竞品风险反推

| 风险 | 来自竞品经验 | 对策 |
| --- | --- | --- |
| 缺少戏剧性 | 没有击杀/投票可能显得平淡 | 强化接触结算反馈、坏人行动提示、终局猜分逆转。 |
| 讨论太模板 | 结构化按钮可能降低表演空间 | Demo 先保证可用，后续加入自由文本或更多声明类型。 |
| 记忆负担 | 分数和接触链容易忘 | 局内日志 + 终局复盘是 P0。 |
| AI 不像人 | 社交推理的人味来自动机和撒谎 | 给 AI 加性格参数：保守、激进、搅局、跟风。 |
| 平衡不可解释 | 胜率偏差但找不到原因 | 每局记录接触次数、拒绝次数、阵营平均分和猜分影响。 |

## 6. Demo 对外介绍稿

《负一和一》是一款以分数为核心的类狼人杀原型。玩家不通过投票淘汰别人，而是每回合选择一名对象进行接触：好人与好人接触会共同加分，负一与好人接触会污染对方分数。所有人都可以在讨论阶段声明、指控或辩护，但这些信息未必真实。终局时，玩家不仅要争取高分，还要猜出其他人的真实分数。这个设计把狼人杀里的“身份推理”转化为“收益、风险和谎言”的连续博弈，并通过 AI 批量测试支持系统调参。

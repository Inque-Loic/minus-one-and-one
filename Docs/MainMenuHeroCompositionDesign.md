# 《负一和一》主界面 Hero 构图设计文档

更新时间：2026-06-11  
目标界面：主菜单 / 标题页  
用户新方向：参考 JRPG / 独立游戏标题页构图，文字不放中心，画面需要有突出的主体。

## 1. 设计方向修正

上一版方案偏“中央信息面板”，适合稳妥发布，但不够有主视觉。  
新方案改为：

> 偏置标题菜单 + 大主体插画 + 桌游推理符号

也就是让画面像一张游戏标题插画，而不是一个居中 UI 面板。

参考图的共同点：

| 参考特点 | 可借鉴点 |
| --- | --- |
| 主体占据画面一侧或大面积中心偏侧 | 玩家第一眼先看到角色/塔/Logo，而不是按钮。 |
| 菜单偏右或偏下 | 功能入口不抢主体，但清楚可点。 |
| 标题与主体形成对角线 | 画面有动势，不呆板。 |
| UI 文字少，背景承担氛围 | 主菜单更像封面。 |
| 画面有亮暗分区 | 文字放在较暗或干净区域，保证可读。 |

## 2. 核心构图建议

推荐采用 **“左侧标题菜单 + 右侧主体奇观”**。

原因：

- 当前背景右侧已有日光、仙山宫殿、塔阁，是天然主视觉区域。
- 左侧中下有棋盘和暗部，适合放标题和菜单。
- 这样能避免标题继续压在画面中心，也不会挡住远景宫殿。

### 3.2 视觉重心

| 区域 | 权重 | 内容 |
| --- | --- | --- |
| 右侧 55%-65% | 主视觉 | 仙山宫殿、远景日光、漂浮卡牌、棋子对峙。 |
| 左侧 25%-35% | 信息区 | 标题、副标题、菜单按钮。 |
| 左下角 | 桌游锚点 | 棋盘、筹码、卡牌保留并略微提亮。 |
| 右下角 | 风险锚点 | 可加红色卡背或负一棋子，呼应阵营对立。 |

## 4. 主体设计

当前背景没有角色立绘，因此“突出主体”不建议硬加人物。更适合从现有题材中提炼一个主体：

> 云海棋局中的“对峙身份牌 / 红绿棋子 / 仙山宫殿”

### 4.1 主体元素组合

| 元素 | 位置 | 作用 |
| --- | --- | --- |
| 远景宫殿 | 右上至中右 | 保留为世界观主体。 |
| 一张金绿身份卡 | 右中偏下，透明度 35%-50% | 代表“一阵营”。 |
| 一张赤红身份卡 | 右下或宫殿前景，透明度 35%-50% | 代表“负一阵营”。 |
| 绿色棋子与红色棋子 | 右下塔阁附近或中右云海上 | 明确阵营对立。 |
| 淡金阵法圆环 | 宫殿后或标题附近，透明度 8%-12% | 统一推理/仪式感。 |

主体不需要过多细节，重点是形成“右侧有东西可看”的封面感。

### 4.2 主体优先级

如果只能做一版，优先做：

1. 右侧宫殿区域提亮和保留。
2. 右下加入红绿棋子对峙。
3. 右中加入两张半透明卡背。

不要为了主体添加全新复杂角色立绘，除非后续专门制作角色设定。

## 5. 标题和菜单位置

### 5.1 标题区

推荐位置：

- X：屏幕宽度 18%-28%
- Y：屏幕高度 36%-46%
- 对齐：左对齐

标题内容：

```text
负一和一
```

副标题建议改为：

```text
云上疑局
接触、声明与猜分
```

说明：

- “云上疑局”更短，更像标题页副标。
- “接触、声明与猜分”保留玩法核心。
- 不再使用长句压在中心。

### 5.2 菜单区

推荐位置：

- 位于标题下方，左对齐。
- 两个按钮纵向排列。
- 不使用大面积按钮框，可以改为“文字菜单 + 小图标 + 选中高亮”。

菜单建议：

```text
▶ 开始游戏
● 游戏测试
```

或更轻量：

```text
开始游戏
游戏测试
```

主按钮状态：

- 当前选中/默认项“开始游戏”：金色文字 + 左侧小箭头/棋子。
- 次要项“游戏测试”：青灰文字 + 较低亮度。

## 6. 两种落地版本

### 版本 A：偏置标题 + 右侧主体，推荐

特点：最接近参考图的标题页感。

实现：

1. 移除中央半透明大面板。
2. 标题和菜单整体移到左侧。
3. 左侧标题区背后加局部暗色渐变，保证文字清晰。
4. 右侧保留宫殿和日光作为主视觉。
5. 右下加红绿棋子、卡牌或筹码作为桌游主体。

优点：

- 构图更像正式游戏标题页。
- 不遮挡背景主体。
- 信息层级自然。

风险：

- 左侧文字如果没有暗化承托，可能被背景吃掉。

### 版本 B：右侧菜单 + 左侧主体

特点：更像部分 JRPG 标题页。

实现：

1. 左侧突出棋盘、卡牌、阵法作为主体。
2. 右侧放标题和菜单。
3. 右侧菜单利用天空/云雾干净区域排列。

不推荐作为首选：

- 当前背景右侧日光较亮，文字对比会更难处理。
- 右侧已有宫殿主体，菜单放右侧会抢画面。

### 版本 C：大 Logo 左中 + 菜单左下 + 右侧宫殿主体

特点：更接近《Slay the Spire》参考图。

实现：

1. 标题变成更大 Logo，放左中。
2. 菜单放标题下方，纵向文字列表。
3. 右侧宫殿和塔阁作为主体。

适合后续如果要制作正式 Logo 时使用。

## 7. 具体 UI 规格

### 7.1 背景遮罩

不要全屏压暗太多，改用局部暗化：

| 区域 | 处理 |
| --- | --- |
| 左侧标题区 | 添加从左到右的暗色渐变，透明度 35%-50%。 |
| 中央云海 | 保持较亮，不放文字。 |
| 右侧宫殿 | 保持亮度，作为主体。 |
| 底部桌面 | 轻微提亮 5%-10%，让桌游元素可见。 |

### 7.2 标题

| 项目 | 建议 |
| --- | --- |
| 字号 | 64-82 px |
| 颜色 | 暖金 `#F2C45A` |
| 描边 | 深青黑描边 2-3 px |
| 阴影 | 黑绿阴影，透明度 60% |
| 对齐 | 左对齐 |

标题不需要放进框里。要让它像“Logo”一样压在画面上。

### 7.3 副标题

| 项目 | 建议 |
| --- | --- |
| 字号 | 24-30 px |
| 颜色 | 米白或浅青白 |
| 行数 | 2 行 |
| 对齐 | 左对齐 |
| 间距 | 标题下 12-18 px |

### 7.4 菜单

| 项目 | 建议 |
| --- | --- |
| 形式 | 文字菜单优先，弱化大按钮框 |
| 字号 | 30-36 px |
| 行距 | 14-20 px |
| 默认高亮 | 开始游戏 |
| 高亮颜色 | 金色 |
| 普通颜色 | 青白 |
| Hover/选中 | 左侧出现小箭头、棋子或横线 |

如果保留按钮底图，按钮应更窄、更轻，不要像居中大按钮那样抢主体。

## 9. 元素增删清单

### 删除 / 移除

| 元素 | 原因 |
| --- | --- |
| 中央标题和按钮组 | 不符合“文字不要放中心”的目标。 |
| 中央大半透明方框 | 会削弱背景主体，像占位 UI。 |
| 过多问号/感叹号装饰 | 会让主界面像教程页，减少封面感。 |

### 保留

| 元素 | 原因 |
| --- | --- |
| 背景仙山和宫殿 | 当前最有视觉价值的主体。 |
| 左下棋盘和筹码 | 强化桌游感。 |
| 两个入口 | 当前 Demo 功能简单，保持清楚。 |

### 新增

| 元素 | 作用 |
| --- | --- |
| 左侧暗色渐变承托 | 保证标题/菜单可读。 |
| 右侧红绿棋子对峙 | 提供明确主体和阵营主题。 |
| 右侧半透明身份卡 | 强化“身份推理”。 |
| 小版本号 | 发布感和完整度。 |

## 10. 最终推荐稿

推荐最终画面：

```text
左侧暗部：

负一和一
云上疑局
接触、声明与猜分

▶ 开始游戏
  游戏测试

右侧亮部：

仙山宫殿作为大主体
云海中漂浮一绿一红两张身份卡
右下角红绿棋子对峙，附近点缀筹码
```

这个版本最符合你的新要求：

- 文字不在中心。
- 画面右侧有突出主体。
- 保留当前背景优势。
- 菜单像正式游戏标题页，而不是功能面板。
